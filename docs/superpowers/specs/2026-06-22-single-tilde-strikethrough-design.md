# 단일 틸드 취소선 오작동 수정 설계

- 날짜: 2026-06-22
- 브랜치: `fix/single-tilde-strikethrough`

## 문제

표/텍스트에서 `~`를 범위 구분자로 쓴 경우(`0.4~1.0`, `30~300s`, `0~9`, `1~3`),
BlockNote의 `tryParseMarkdownToBlocks`가 내부적으로 사용하는 `remark-gfm`이
**단일 틸드**(`~...~`)도 취소선으로 해석한다(`singleTilde: true`가 기본값).

그 결과 두 `~` 사이 구간 전체가 취소선으로 렌더링된다.
예: `(0.4~1.0), 저배터리 시 홈 복귀, ... 대기 시간(30~300s)`에서
첫 `~`부터 다음 `~`까지가 통째로 취소선 처리된다.

참고로 GitHub의 실제 렌더링은 `~~이중 틸드~~`만 취소선으로 처리하므로,
현재 동작은 GitHub 표준과도 어긋난다.

## 목표 동작 (결정: 옵션 A)

- 단일 틸드(`~`)는 **절대 취소선이 아니다.**
- 오직 `~~이중 틸드~~`만 취소선으로 처리한다 (GitHub/CommonMark 표준에 맞춤).
- 범위 표기 `0.4~1.0` 등은 자연스럽게 일반 텍스트가 된다.

## 제약

- BlockNote 0.49의 `tryParseMarkdownToBlocks`는 `remark-gfm`의 `singleTilde`
  옵션을 외부로 노출하지 않는다. 따라서 옵션을 직접 끌 수 없다.
- BlockNote 공식 문서가 권하는 우회는 "직접 unified 파이프라인으로 HTML 변환 후
  `tryParseHTMLToBlocks` 사용"이지만, 이는 의존성 추가 + 로드 파이프라인 전면 교체로
  잘 동작 중인 KaTeX(```math)·Mermaid 블록 처리를 깨뜨릴 위험이 크다 → 채택하지 않음.

## 접근 (채택)

**로드 시점에 단일 틸드를 이스케이프한다.**

`tryParseMarkdownToBlocks`에 넘기기 직전, 파일에서 읽은 마크다운 문자열에서
단일 `~`를 `\~`로 변환한다. `~~`(이중 틸드)는 건드리지 않아 의도된 취소선은 보존된다.
파싱 후 블록 텍스트는 백슬래시가 이스케이프로 소비되어 깔끔한 `0.4~1.0`이 되고,
취소선이 생기지 않는다.

장점:
- 의존성 추가 없음.
- 변경 범위가 작다.
- 기존 KaTeX/Mermaid/라운드트립 파이프라인을 건드리지 않는다.

## 구현

### 1. `escapeSingleTildes(md: string): string` (신규, `customParse.ts`)

코드 영역을 건너뛰며 단일 틸드만 이스케이프한다.

1. **라인 단위로 코드 펜스 추적** — `^(\s*)(`{3,}|~{3,})` 로 펜스 열림/닫힘을 감지.
   펜스 내부 라인은 절대 변환하지 않는다(`~~~` 펜스 마커 자체도 보존).
   닫힘 펜스는 동일 문자(` ` `` 또는 `~`)이고 길이가 여는 펜스 이상이어야 한다.
2. **펜스 밖 라인은 인라인 코드 스팬 분리** — `` `...` `` 코드 스팬은 그대로 두고,
   나머지 텍스트에서만 치환한다.
3. **치환 규칙** — `/(?<!\\)(?<!~)~(?!~)/g` → `\~`
   - `(?<!\\)` : 이미 이스케이프된 `\~` 제외(이중 이스케이프 방지)
   - `(?<!~)~(?!~)` : 단일 틸드만 매칭(양옆이 `~`가 아닌 경우) → `~~취소선~~` 보존

### 2. 로드 경로 적용 (`Editor.tsx`)

초기 로드(현재 L40)와 focus 재로드(현재 L120) 두 곳에서 파싱 직전에 적용:

```ts
const blocks = await editor.tryParseMarkdownToBlocks(escapeSingleTildes(md));
```

중복을 피하기 위해 작은 헬퍼로 묶는 것을 고려한다.

### 3. 라운드트립 안전장치 (검증 필요)

로드 후 블록 텍스트는 깔끔한 `0.4~1.0`이다. 저장 시 `blocksToMarkdownLossy`가
`~`를 다시 `\~`로 이스케이프해 파일을 더럽히는지 **회귀 테스트로 확인**한다.
- 더럽히지 않으면: 추가 작업 없음(YAGNI).
- 더럽히면: 저장 출력에서만 `\~` → `~` 정규화를 추가한다.

## 테스트 (TDD)

`src/markdown/__tests__`에 추가:

- `0.4~1.0` → `0.4\~1.0` (단일 틸드 이스케이프)
- `~~취소선~~` → 변경 없음 (이중 틸드 보존)
- 코드 펜스(` ``` ` 및 `~~~`) 내부의 `0.4~1.0` → 변경 없음
- 인라인 코드 `` `0.4~1.0` `` → 변경 없음
- 이미 이스케이프된 `\~` → 이중 이스케이프 안 함
- 표 전체 라운드트립: 로드 → 저장 후 `0.4~1.0` 유지 & 취소선 마크업 없음

- 단일 틸드 취소선을 옵션으로 재활성화하는 설정.
- `~~~` 3개 초과 등 비정상 펜스의 정밀 처리.
